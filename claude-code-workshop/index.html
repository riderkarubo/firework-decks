Ch.1 初期設定・最初にやること

⏱ 20分 ハンズオン
🎯 このセクションのゴール
全員が Claude.ai にログインし、/setup-cowork のガイドに沿って自分のロールに合った環境を整える。
  • 1
    claude.ai にログインして Cowork を開く
    ブラウザで claude.ai にアクセスしてログイン。左メニューの「Cowork」をクリック。
    ※ Cowork は Pro プラン（月$20）以上が必要
  • 2
    /setup-cowork を実行する
    チャット入力欄に /setup-cowork と入力して送信。
    Anthropic 公式のガイド付きセットアップが始まる。以下の5ステップを画面に沿って進める。
    1️⃣
    ロール確認
    職種（マーケター・クリエイティブ・営業など）を選ぶ。ロールに合った設定を自動提案してくれる。
    2️⃣
    プラグインの提案
    ロールに合ったプラグイン（スキル＋コネクターのセット）をマーケットプレイスから検索・提示。インストールするだけで /キャンペーン企画 のようなワークフローが使えるようになる。
    3️⃣
    コネクターの確認・追加
    接続済みのツール（Slack・Gmail・Google Calendar など）を確認。プラグインが必要とする未接続ツール（Canva など）をConnect画面付きで提案してくれる。
    4️⃣
    スキルをひとつ試す
    インストールしたプラグインのスキルを一覧表示。その場で実行して「こう使うのか」を体験できる。
  • 3
    Claude の設定を一緒に確認する
    右上のアイコン →「Settings」を開く。2つのタブを順番に見ていく。
    ⚙️
    Claude の設定（AI 挙動）
    Default model：タスクに合わせて変えられる（重い作業は Opus、速さ重視は Haiku）。
    Enable Artifacts：コードや HTML を別パネルで表示する機能。ON 推奨。
    Extended thinking：複雑な問題を深く考えさせるモード。重い分じっくり使いたい時だけ ON。
    🔌
    Integrations（連携ツール）
    Slack・Gmail・Google Drive・Calendar など、すでに繋がっているツールを確認。
    「+ Connect」 から追加できる。繋いでおくと Claude がツールをまたいで動ける。
    まず自分のアカウントで何が繋がっているか見てみよう。
💡
Integrations を繋ぐと何が変わる？
たとえば Google Drive を繋ぐと、「先月の企画書を要約して」と言うだけで Claude がドライブを検索して答えてくれる。 ツールをまたいだ作業が「指示1回」で完結するようになる。
⚠️
大事な作業を任せる前に必ず送る「保険プロンプト」
「ファイルの削除は禁止・取り消せない操作は承認を取ること。これを守ってから始めて。」
Integrations を有効にした Claude はツールをまたいで動けるので、大きな作業の前に毎回確認させる癖をつけよう。

Ch.2 フォルダ構造を見る

⏱ 15分 デモ
📁 Cowork はプロジェクト単位でファイルを「覚えて」くれる
フォルダ・ドキュメント・台本を読み込ませると、Claudeがその文脈を持った状態で会話してくれる。
「毎回説明しなくていい」パートナーになる。

制作プロジェクトのフォルダ例（GIFTVOX 向けライブ配信案件）

📁 PROJECT_クライアント名/ ├── CLAUDE.md ← AIへの指示書（ここが超重要） ├── 📁 01_企画・台本/ │ ├── 企画書_v1.md │ └── 台本_配信用_260528.md ├── 📁 02_素材/ │ ├── 商品スペック.xlsx │ └── 過去配信メモ.txt ├── 📁 03_成果物/ │ └── 配信報告書.html └── 📁 04_メモ/ └── クライアント打ち合わせメモ.md
⭐ CLAUDE.md が最強の武器
プロジェクト直下に CLAUDE.md というファイルを置くと、Claudeがそこを最初に読んでくれる。

書くべき内容：
・クライアント名・案件概要
・よく使う言葉・表記ルール
・「これをやるときはこうして」という行動ルール
・禁止事項（「弊社」じゃなくて「Firework」と書いて、など）
💡
コツ：最初にプロジェクトを説明させる Coworkに「このプロジェクトフォルダを見て、何の仕事か説明して」と聞いてみると、ちゃんと読み込んでいるか確認できる。
⭐
作業を任せる前に「要件を先に伝える」が鉄則 いきなり「やって」と言わず、先に「こういう成果物を作りたい」と説明して、Cowork に「どんな方針で進めるか」を確認させてから実行させる。
例：「5つのExcelデータから提案資料を作りたい。どんな構成で作ればいいか聞いてから進めて」→ アウトラインの確認 → OKを出してから実行

Ch.3 Chat / Cowork / Code の使い分け

⏱ 15分 ハンズオン
💬
Chat
ひとこと相談
単発の質問・アイデア出し・文章校正。過去の会話は引き継がれない。
例：「この件名どう思う？」「3つアイデア出して」「誤字チェックして」
🗂️
Cowork
プロジェクト管理
ファイルを読み込み・記憶を持ったまま長期間の仕事をする。制作業務の主役。
例：「この企画書を読んで台本を書いて」「前回のFBを踏まえて修正して」
⚡
Code
自律実行
ファイルの作成・編集・削除・コマンド実行を自動でやってくれる開発者モード。
例：「この100件のCSVを整理して」「レポートHTMLを自動生成して」
🤔 どれを使うか迷ったとき の判断フロー
ちょっと聞きたいだけ・単発の質問
→
Chat
案件のファイルを読んで長期間作業してほしい
→
Cowork
大量データ処理・自動生成・コード書いてほしい
→
Code
迷ったら
→
Cowork を選ぶ
✨
GIFTVOX チームへのおすすめ ライブ配信・動画制作の仕事は Cowork が主役になる。台本・企画書・過去メモをプロジェクトに入れておけば、「前回どんな演出だったっけ？」も即答してくれる。
💰
トークン消費量の違い：シンプルな作業は Chat で十分 Cowork はエージェントとして動くためトークン消費が多め。単純な質問・文章校正・アイデア出しなど「ファイル操作が不要な作業」は Chat で済ませるとコスト節約になる。
目安：Chat＝通常会話、Cowork＝ファイルを読み書きする仕事、Code＝ターミナル操作が必要な開発作業

Ch.4 モデルの使い分け

⏱ 10分 講義

Claude には複数のモデルがある。「速さ・コスト・賢さ」のバランスで選ぶ。

Haiku
⚡⚡⚡ 最速・最安
シンプルなQ&A・翻訳・要約・短い文章整理に最適。
使う場面：毎日使うルーティン作業・返答を急ぐ場面
Sonnet
⚡⚡ バランス型（デフォルト）
企画書・台本・提案書・分析レポートなど業務のメイン。
使う場面：通常の制作・文章作成のほぼすべて
Opus
⚡ 深い思考・高精度
複雑な戦略策定・精緻なロジック・難しい分析に使う。
使う場面：重要提案書の仕上げ・複雑な問題解決
Effort（Extended Thinking）
🧠 じっくり考えるモード
モデルが「考える時間」をかけて精度を上げる。答えが出るまで少し待つ。
使う場面：「答えがすぐ出なくていいから正確に」という場面
⚠️
最初はSonnetだけ使えばOK 迷ったらデフォルトの Sonnet で十分。慣れてきたら「これはHaikuで十分だな」「ここはOpusで念入りに」と使い分けるようになる。

Case Study 01 — コープこうべ

ワイン試飲イベント 予告編企画

ライブ配信の企画アイデアを Claude Cowork で生成・ブラッシュアップ
📌 やったこと
コープこうべのワイン試飲イベントの 予告編配信企画案 を Cowork で作成。
クライアントのブランドトーン・過去配信メモ・対象視聴者情報をプロジェクトに読み込ませ、 複数の企画アイデアを生成 → Firework 内でブラッシュアップ → クライアントに HTML 形式で提出。
1️⃣
プロジェクト準備
Cowork にクライアント情報・過去メモ・ターゲット像を入力。CLAUDE.md に「コープこうべのトーンは親しみやすく」と指定。
2️⃣
企画アイデア生成
「ワイン初心者が見たくなる予告編企画を5案出して」→ 複数の切り口から提案が出てくる。
3️⃣
フィードバックで絞り込み
「3案目の方向性が好き。もっと集客フックを強くして」→ Cowork が文脈を持ったまま修正。
4️⃣
HTML提案書として出力
「これを見やすいHTML提案書にして」→ そのままクライアントに共有できるドキュメントが完成。
⭐
ポイント：「文脈を持ったまま修正できる」が最大の武器 Chat だと毎回ゼロから説明しなければならないが、Cowork はプロジェクトの文脈を持ち続けるので「さっきの3案目をもっと〇〇にして」が通じる。
Case Study 02 — データ分析・スライド作成

配信数値を分析してレポートを自動生成

CSVデータ → インサイト → スライド資料 を Claude で一気通貫
📌 やったこと
配信後の数値データ（視聴数・CTR・コメント率 etc.）を CSV で Cowork に渡し、
分析インサイトを抽出 → クライアント向けレポートHTMLを自動生成。
これまで数時間かかっていた作業を大幅に短縮。

Ch.6 Computer Use とは？

⏱ 10分 デモ
🖥️ AIが「目で見て手を動かす」
Computer Use は、Claude がスクリーンショットを見ながらマウス・キーボードを操作できる機能。
「人間がやっていたPC作業をそのまま自動化」できる。
🔍
できること
ブラウザでフォームに入力・スプレッドシートを操作・アプリ間でコピペ・リサーチして情報収集
🚫
注意点
まだ精度は発展中。「確認しながらやる」使い方が現実的。完全自律よりはパートナー感覚で。
🎬
制作業務での活用場面
複数ツール間のデータ転記・SNS投稿チェック・競合アカウントのリサーチ自動化
🔒
セキュリティ
パスワード入力・個人情報が含まれる操作は慎重に。業務用PCで使う場合は会社ルールを確認。
🎥
デモ：Claude がブラウザを操作する様子を見てみよう ここで実演デモを行います。「このURLのページのタイトルと要約を集めて」と指示する例を見せます。

Ch.7 ハンズオン：実際にやってみよう

⏱ 10分 ハンズオン
🎯 お題
自分が担当している（または担当したことがある）案件や業務を1つ選んで、
Cowork に投げてみましょう。
📋 ハンズオン チェックリスト
Claude.ai にログインして Cowork（Projects）を開く
アカウントがない人は Chat でもOK
新しいプロジェクトを作成する
名前は「ハンズオン_自分の名前」でOK
案件の背景を Cowork に説明する
「私は○○の会社でライブ配信の制作をしています。今月は〇〇というクライアントの〇〇という企画を担当しています」など
業務の中で時間がかかっている作業を1つ投げてみる
例：台本の構成・クライアントへのメール文・企画書のたたき台・報告書フォーマット
返ってきた答えにフィードバックして修正させる
「もっとカジュアルなトーンで」「この部分をもう少し具体的に」など
隣の人と結果をシェアする
どんなプロンプトを入れたか・どんな返答が来たかを話してみよう
🗣️
プロンプトのコツ：3要素を入れると精度が上がる ①自分は誰か（役割） ②相手は誰か（読み手） ③何をしてほしいか（動詞で具体的に）

例：「私はライブ配信ディレクターです。クライアントに送る配信後の報告メール（200字程度）を書いてください。今月の配信は視聴者3,000人・CTR1.8%でした。」

+α 台本作成の業務効率化（時間があれば）

⏱ 5分 事例
🎬 ライブ配信台本を Cowork で半自動化
商品スペック・過去配信メモ・ターゲット像をプロジェクトに入れておくと、
「〇〇の台本を作って」と一言で叩き台が出来上がる。

GIFTVOXで使える例：
・商品紹介ライブの台本（オープニング〜クロージングまで）
・動画ナレーション原稿
・YouTube概要欄・ハッシュタグ生成
# 台本生成プロンプトの例 「以下の情報をもとに、60分のライブ配信台本を作ってください。 商品：〇〇（価格・スペック・ターゲット層） 配信スタイル：2人掛け合い、テンポよく 見どころ：実演デモ・限定価格の告知 クロージング：視聴者への感謝＋次回予告 構成：オープニング(5分)→商品紹介(20分)→実演(25分)→Q&A(7分)→クロージング(3分)」
