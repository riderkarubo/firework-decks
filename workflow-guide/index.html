STEP 01

素材を整理する

1
AIに渡す素材をテキストでまとめる
以下の情報を箇条書きで用意してください。完璧でなくても大丈夫です。
  • 配信・イベントの目的（例：ワイン試飲イベントへの申込を増やしたい）
  • ターゲット（例：30〜40代、夜19時視聴）
  • 出演者と各自の役割・特徴
  • 過去のデータや実績（例：昨年の視聴分数、盛り上がったコーナー）
  • 制約・NGなこと（例：商品購入はゴールにしない）
ポイント：過去の配信データや文字起こしがあれば、そのまま貼り付けて渡してください。AIが自分で読み解いて使えるデータを拾ってくれます。
STEP 02

AIに企画案を出してもらう

2
プロンプトのテンプレート
以下をコピーして、【】の部分を書き換えてAIに送ってください。
AIへのプロンプト（コピーして使う）
【クライアント名】向けの【配信・イベント名】について、企画案をHTMLにまとめてほしいです。

■ 目的
【例：ワイン試飲イベントの申込者を増やすための予告編配信】

■ ターゲット
【例：30〜40代女性、夜19時視聴層】

■ 出演者
【例：ソムリエ（専門家）、羽田さん（視聴者代弁役）、平林さん（飲めない担当）】

■ 過去データ・参考情報
【例：昨年の視聴分数は11分52秒で歴代上位だったが購入ゼロ。300名応募の実績あり】

■ デザインのイメージ
マガジン風のHTMLで、セクションごとに企画カードを並べてください。
カラーは #fa006d（赤ピンク）をアクセントに使ってください。
各カードには通し番号（No.1, No.2...）をつけてください。

■ 構成イメージ
・カバー（タイトル・ターゲット・出演者）
・設計思想（なぜこの企画か）
・出演者の役割メモ
・セクション別の企画カード（3〜4セクション）
・フッター
Coworkを使う場合：新規チャットを開き、上のプロンプトをそのまま貼り付けてください。添付ファイルがあれば一緒に渡すとより精度が上がります。
3
出力されたHTMLをファイルに保存する
AIが出力したHTMLを、テキストエディタ（VSCode など）にコピーして index.html として保存してください。
ブラウザで開いて表示を確認します。
  • Mac：ファイルをダブルクリック → Chromeなどで開く
  • 表示が崩れていたらAIに「レイアウトが崩れています」と伝えると直してもらえます
STEP 03

内容を編集・ブラッシュアップする

4
AIとの対話で内容を磨く
同じチャットで続けて指示を出すと、AIがHTMLを修正してくれます。以下のような指示が使えます。
編集指示の例
「No.3は単体では機能しないのでNo.5に統合してください」
「セクション2を丸ごと削除してください」
「No.1の文章を修正してください。〇〇さんはワインを飲まないので、おつまみ担当として書いてください」
「各カードの通し番号を上から順に振り直してください」
「セクション3と4の順番を入れ替えてください」
カードを削除・並び替えたあとは「通し番号を上から No.1, No.2... に振り直してください」と必ず伝えてください。番号がずれたままだとクライアントが混乱します。
5
完成したら最終チェック
  • プレースホルダー（〇〇、TBD、XXX）が残っていないか
  • カードの通し番号が上から順になっているか
  • クライアント名の表記が統一されているか
  • 読んで違和感のある文章がないか
STEP 04

パスワードをかけて公開する

6
パスワードを決めてハッシュ値を生成する
パスワードは平文のままファイルに書かず、SHA-256ハッシュを使います。
Macのターミナルで以下を実行してください（パスワードの部分は変えてOK）。
# パスワードを決める（英数字推奨）
PW="Coop-AbCd1234"
echo "パスワード: $PW"
printf '%s' "$PW" | shasum -a 256 | cut -d' ' -f1
ターミナルが使えない場合：AIに「パスワード "Coop-AbCd1234" のSHA-256ハッシュ値を教えてください」と聞けば計算してもらえます。
7
gate.js をフォルダに追加する
以下の内容で gate.js を作成し、index.html と同じフォルダに置いてください。
PASSWORD_HASH の部分を、STEP 6で生成したハッシュ値に書き換えます。
(function () {
  var PASSWORD_HASH = 'ここにSHA-256ハッシュ値を貼る';
  var STORAGE_KEY = '案件名_auth'; // 案件ごとに変える
  var TITLE = '企画資料タイトル';
  var SUBTITLE = 'この資料はパスワードで保護されています';

  // ↓以下はそのままコピー。変更不要
  function sha256Hex(text) {
    var enc = new TextEncoder();
    return crypto.subtle.digest('SHA-256', enc.encode(text)).then(function (buf) {
      var bytes = new Uint8Array(buf), hex = '';
      for (var i = 0; i < bytes.length; i++) hex += bytes[i].toString(16).padStart(2, '0');
      return hex;
    });
  }
  function reveal() { var g = document.getElementById('fw-gate'); if (g) g.remove(); document.body.style.overflow = ''; }
  function buildGate() {
    document.body.style.overflow = 'hidden';
    var w = document.createElement('div'); w.id = 'fw-gate';
    w.innerHTML = '<div class="fw-gate-card"><div class="fw-gate-icon">🔒</div><h1>'+TITLE+'</h1><p>'+SUBTITLE+'</p><form id="fw-gate-form" autocomplete="off"><input id="fw-gate-input" type="password" placeholder="パスワードを入力" autofocus><button type="submit">閲覧する</button></form><div id="fw-gate-err" class="fw-gate-err"></div></div>';
    document.body.appendChild(w);
    var s = document.createElement('style');
    s.textContent = '#fw-gate{position:fixed;inset:0;z-index:99999;background:#1A1A2E;display:flex;align-items:center;justify-content:center;font-family:"Noto Sans JP",sans-serif;padding:24px;}.fw-gate-card{background:#fff;border-radius:14px;padding:40px 36px;max-width:380px;width:100%;text-align:center;box-shadow:0 10px 40px rgba(0,0,0,.4);}.fw-gate-icon{font-size:40px;margin-bottom:14px;}.fw-gate-card h1{font-size:17px;font-weight:700;color:#1A1A2E;margin:0 0 6px;}.fw-gate-card p{font-size:12px;color:#888;margin:0 0 22px;}#fw-gate-form{display:flex;flex-direction:column;gap:10px;}#fw-gate-input{padding:12px 14px;border:1px solid #ccc;border-radius:8px;font-size:14px;outline:none;}#fw-gate-input:focus{border-color:#FA006D;}#fw-gate-form button{padding:12px;border:none;border-radius:8px;background:#FA006D;color:#fff;font-size:14px;font-weight:700;cursor:pointer;}.fw-gate-err{color:#FA006D;font-size:12px;margin-top:12px;min-height:16px;}';
    document.head.appendChild(s);
    document.getElementById('fw-gate-form').addEventListener('submit', function (e) {
      e.preventDefault();
      sha256Hex(document.getElementById('fw-gate-input').value).then(function (hex) {
        if (hex === PASSWORD_HASH) { try { localStorage.setItem(STORAGE_KEY, '1'); } catch (_) {} reveal(); }
        else { document.getElementById('fw-gate-err').textContent = 'パスワードが正しくありません。'; document.getElementById('fw-gate-input').value = ''; }
      });
    });
  }
  function init() { var ok = false; try { ok = localStorage.getItem(STORAGE_KEY) === '1'; } catch (_) {} if (!ok) buildGate(); }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init); else init();
})();
8
index.html に gate.js を読み込む2行を追加する
index.html の <head> の最初の方（<meta charset> の直後）に以下を追加してください。
<meta name="robots" content="noindex, nofollow">
<script src="./gate.js"></script>
../auth.js のような別の認証ファイルが元から書かれている場合は、それを削除してから上の2行を追加してください。
STEP 05

GitHub Pagesで公開する

9
フォルダ構成を確認する
公開するフォルダに以下の2ファイルが入っていればOKです。
  • index.html — 企画案本体
  • gate.js — パスワードゲート
10
GitHubリポジトリにアップロードする
  • GitHub.com でリポジトリを作成（Public に設定）
  • 2ファイルをアップロード（「Add file → Upload files」）
  • Settings → Pages → Branch: main / folder: / (root) → Save
  • 数分後に公開URL（https://ユーザー名.github.io/リポジトリ名/）が発行される
gh CLIが使える場合：以下のコマンドで一気に完了します。
git init -b main && git add . && git commit -m "初版" && gh repo create リポジトリ名 --public --source=. --remote=origin --push
その後 gh api -X POST repos/ユーザー名/リポジトリ名/pages -F "source[branch]=main" -F "source[path]=/"
11
公開後の動作確認
URLにアクセスしてパスワード画面が表示されることを確認します。
  • パスワードを入れて企画案が表示されることを確認
  • ブラウザを閉じて再アクセスしてもパスワード不要になっていることを確認
  • 別のブラウザ・スマホからはパスワードが必要なことを確認
STEP 06

クライアントに送る

12
URLとパスワードは別経路で送る
URLとパスワードを同じメッセージに書くとパスワードの意味が薄れます。
  • URL → Slack や メール本文
  • パスワード → 別のメッセージ、または口頭で伝える
13
Slackメッセージの文面（参考）
AIに「クライアント向けのSlackメッセージを作ってほしい」と頼むと、適切なトーンで書いてくれます。以下のように伝えると精度が上がります。
AIへの指示例
○○さん（クライアント）に企画資料を共有するSlackメッセージを作ってください。
URL：https://...
パスワードは別送します。
企画を考えたポイント（3点）も添えてください。フォーマルなトーンで。
メッセージ内でAIが使いがちな「〇〇の「」」形式の太字（アスタリスク2個で囲む表現）は削除してください。読み手にとって不自然に見えます。

パスワードゲートが表示される（URLにアクセスして確認）
